All or Nothing

A transaction is a single unit of work. If any part of the transaction fails, the entire transaction fails, and the database state is left unchanged.

lightbulb The Concept

Imagine transferring money. It involves two steps: subtracting from one account and adding to another. Atomicity guarantees that you never have a situation where money is subtracted but not added.

check_circle

Commit

All steps succeed. Changes are saved permanently.

replay

Rollback

One step fails. System reverts to original state.

Scenario: Bank Transfer

Alice wants to send $100 to Bob. The system must perform two updates. If the power fails after the first update, Alice loses money, and Bob gets nothing. Atomicity prevents this.
